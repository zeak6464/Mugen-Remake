# Roadmap

Priorities change with contributors and forks; treat this as a **directional** list, not a promise schedule.

## Near term (engine feel + trust)

- **Stability**: fix edge cases in team modes, replay playback, and import regeneration.
- **Documentation**: keep guides in sync with `states.json` / controller additions; more small examples (one “template” character besides bundled content).
- **UX**: consistent menus, clearer first-run path (import → edit → fight).
- **Samples**: maintain one polished reference mod and one minimal “hello fighter” layout.

## Medium term (creator velocity)

- **Validation**: CLI or in-editor checks for JSON/schema errors with actionable messages.
- **Debugging**: frame-by-frame stepping, state timeline overlay, export of last-hit context.
- **Asset pipeline**: smoother material/costume workflow; optional batch tools for audio and VFX hooks.

## Longer term (platform)

- **Networking**: solid rollback or deterministic lockstep is a large effort; current host/join is experimental.
- **Distribution**: export templates, versioning scheme for mod compatibility, optional package format (zip/manifest).

## How to influence the roadmap

Open issues or discussions with: *problem*, *proposed behavior*, and *whether you can contribute a PR*. Engine changes that stay data-driven and mod-compatible are easiest to merge.
